# Slide Prototype Overlay

Use this overlay for presentation prototypes, interactive slide experiments, and visual application demos.

## Rules

- Build the usable slide or prototype surface first, not a marketing page around it.
- Keep generated assets and dependencies opt-in; do not make the generic branch require frontend tooling.
- Verify visual outputs with a rendered screenshot or equivalent inspection when a dev server or browser render is needed.
- Keep text legible at target presentation sizes.
- For academic slides, preserve source-to-claim traceability even when improving visual design.

## Typical Capabilities

- build slide prototypes;
- refine visual hierarchy;
- test rendered layouts;
- create lightweight demo pages;
- adapt slides for talks, teaching, or applications.
